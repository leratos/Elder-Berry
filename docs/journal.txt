### Gekaufte Hardware:
| Teil                                    | Spezifikation                          | Preis   |
|-----------------------------------------|----------------------------------------|---------|
| Raspberry Pi 5 4GB                      | Einzelboard                            | ~86€    |
| RPi Touch Display 2 (5")               | 720×1280, DSI, kapazitiv, Portrait     | ~43,50€ |
| TECNOIOT 2S BMS (5 Stk.)              | 10A Dauer, 20A Limit, Tiefentladeschutz| ~15€    |
| Acrylplatten 150×200×1mm (15 Stk.)    | Transparent, für Pepper's Ghost        | ~9€     |
| microSD 32GB+                           | A2, Class 10 (noch zu kaufen)          | ~10€    |
| **Gesamt**                              |                                        |**~164€**|

### Display-Entscheidung Begründung:
- DSI-Anschluss: GPIO bleibt frei für Adafruit Motor HAT
- 720×1280 Portrait nativ: Perfekt für 512×1024 Sprite, kein Rotation-Hack
- Display-Strom: Separater 5V-Anschluss auf eigener PCB (vom D36V50F5)
- Pepper's Ghost: LCD horizontal, Acryl 45°, schwarzer Hintergrund → Sprite "schwebt"

### Energiebudget:
- 2× 18650 (35A E-Zig-Zellen, ohne Schutz) → 2S BMS → Sicherung 10A → PCB
- D36V50F5: 5V/5.5A für RPi5 + Display (~2-3A normal)
- Motoren: 4× TT-Motor direkt @ 7.4V (~0.8A normal, ~6A Stall)
- Normalbetrieb gesamt: ~3A, Worst Case ~8.5A → Sicherung 10A passt

### Gehäuse:
- Eigenbau: Resin-Druck (12kg Material vorhanden)
- Design in Phase 4 (Inventor), nach Hardware-Prototyp-Test

- Tower = Hirn (LLM + TTS-Generierung, immer an)

## Architektur-Notiz: Remote Tower via NordVPN Meshnet (2026-03-15)
- Laptop unterwegs kann Tower als LLM-Backend nutzen über NordVPN Meshnet
- Meshnet gibt feste Mesh-IPs (100.x.x.x), kein Port-Forwarding nötig
- Voraussetzung Tower: OLLAMA_HOST=0.0.0.0 + Firewall nur Mesh-IP auf Port 11434
- LLMRouter Config: ollama.host = Mesh-IP (unterwegs) oder localhost (zuhause)
- Idee: Auto-Erkennung in LLMRouter (localhost erreichbar? → lokal, sonst → Mesh-IP)
- Latenz unterwegs: 30-80ms (gutes Netz), 150-500ms (schlechtes Netz)
- Streaming-Tokens über Mesh merkbar langsamer, aber funktional
- Fallback: Laptop hat eigenes Ollama (phi4:14b mit RAM-Auslagerung)
- Aktueller Entwicklungsbetrieb: alles auf Laptop, Tower wird später scharfgeschaltet

## Hardware-Entscheidung: Motor-Upgrade TT → JGB37-520 (2026-03-15)
- Grund: Gehäuse 1.138g + Elektronik ~760g + Chassis ~400g = ~2.3kg → TT-Motoren zu schwach
- Neuer Motor: JGB37-520 (12V, Hall-Encoder, 2-10 kg·cm je nach Ratio)
- Traglast mit 4× JGB37: 5-8kg → massig Reserve für ~3kg Roboter
- Encoder ermöglicht PID-Regelung auf Pico 2W
- Konsequenzen:
  - Neues Chassis nötig (JGB37 = Ø37mm Getriebe, deutlich größer als TT)
  - 4× Motor = 784g (statt 120g bei TT) → Gesamtgewicht steigt auf ~2.7-3kg
  - Motor-Driver: Adafruit Motor HAT evtl. zu schwach (1.2A/Kanal, JGB37 Stall = 2.8A)
    → Alternative evaluieren: L298N, BTS7960, oder Custom-Driver auf eigener PCB
  - Akku: 2S (7.4V) läuft, aber JGB37 ist für 12V spezifiziert (60% Drehzahl bei 7.4V)
    → 3S (11.1V) prüfen: BMS-Kompatibilität, D36V50F5 Eingangsbereich, PCB-Redesign
  - Mecanum-Räder: 60mm oder 80mm mit 6mm D-Shaft Aufnahme
- OFFEN: Akku 2S vs 3S → BMS und PCB prüfen
- OFFEN: Motor-Driver Wahl → Phase 4 PCB-Redesign
- OFFEN: Chassis-Design → nach Motor-Bestellung
