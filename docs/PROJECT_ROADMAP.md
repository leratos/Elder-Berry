# Elder-Berry – Project Roadmap

## Phase 1 – Software Basic (Windows Tower)
- PC-Steuerung (Maus, Tastatur, Fenster, Lautstärke)
- Systemdaten auslesen (CPU, RAM, GPU, laufende Prozesse)
- Erste LLM-Anbindung lokal via Ollama
- Aktions-DB aufbauen (Befehl → Aktion, selbstlernend)
- Basis-TTS (Text to Speech)

## Phase 2 – RPi5 Anbindung
- Kommunikationsprotokoll Tower ↔ RPi5 definieren (WLAN/USB)
- Sensor-Integration: Kamera, IR, Temperatur
- RPi5 als I/O-Controller etablieren
- Daten-Pipeline RPi5 → Tower → LLM

## Phase 3 – Display
- Display-Typ definieren (Hologramm/Screen)
- V-Tuber Charakter anlegen (VRoid Studio)
- Lippensync + Emotionen via VTube Studio API
- Charakter auf Display einbinden

## Phase 4 – Körper
- Gehäuse designen (Inventor)
- Mecanum-Roboter anbinden
- Autonomes Laden (Ladestation, Akku-Monitoring)
- Ästhetik + Finish

## Phase 5 – Software Advance
- Vollständige Aktions-Logik ausarbeiten
- Emotion-State-Machine
- Sensor-Kontext in LLM-Prompts integrieren
- OpenRouter Multimodal (Kamera → GPT-4o)
- Feinschliff, Stabilität, Performance
